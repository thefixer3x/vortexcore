# Documentation Index

This index is the entrypoint for all project documentation after root cleanup.

## Structure

- `../plans/` — action plans, rollout plans, roadmaps, checklists
- `../reports/` — audits, findings, completion summaries, performance/security reports
- `../runbooks/` — troubleshooting and incident handling documentation
- `../guides/` — setup and integration guides
- `../testing/` — testing plans and validation documents
- `../architecture/` — architecture and code organization docs
- `../migration/` — migration and impact analysis docs
- `../archive/` — deprecated or historical documents

## Usage Rules

1. New markdown docs should **not** be added at repository root (except main `README.md`).
2. Place docs according to purpose, not author/team.
3. If a document becomes outdated, move it to `docs/archive/` rather than deleting immediately.
4. Prefer linking documents from this index for discoverability.

## Governance

- Root stays clean and operational.
- Categories remain stable over time.
- Documentation updates should include index/link updates where relevant.
