# OBF-001-V — Providus Integration Spike: VortexCore Side

**Date:** 2026-04-26
**Status:** Approved — ready for implementation
**Parent:** OBF-001 cross-repo spike

---

## Goal

Render real Providus accounts and transactions in the VortexCore dashboard via `onasis-gateway`. The existing internal wallet/transaction data source remains untouched and unchanged.

---

## Mental Model

```
VITE_OBF_LIVE=false (default)
  └── Dashboard shows internal wallets + transactions (existing behaviour, untouched)

VITE_OBF_LIVE=true
  └── Dashboard shows OBF section (Providus via gateway) PLUS internal wallets
      — two separate panels, two separate data sources, no merging
```

---

## Two Non-Negotiable Constraints

### 1. No Data Model Merging

Internal wallets and OBF accounts have different schemas, different ownership, and different regulatory posture. Keep them visually and logically separate in the UI.

- An "OBF Accounts" section in the dashboard, not a unified account list
- Internal wallets render in the existing `AccountCard` + `TransactionList` panels
- OBF accounts render in a new dedicated panel
- **No merging now.** Merging creates debt that's expensive to unpick when consent lifecycle (OBF-003) arrives.

### 2. Feature Flag Gates Rendering, Not Fetching

When `VITE_OBF_LIVE=false`:
- The `obf-accounts` edge function is **never called**
- No network request, no fetch attempt, no suppressed response
- Keeps the Providus sandbox clean and avoids burning quota on every dashboard load during development

When `VITE_OBF_LIVE=true`:
- OBF panel renders only when the flag is on
- Edge function called only at that point

## Architecture

### Edge Function: `supabase/functions/obf-accounts`

- **Runtime:** Deno (same as existing edge functions)
- **Secrets (Supabase Vault):** `ONASIS_GATEWAY_URL`, `ONASIS_GATEWAY_TOKEN`
- **Auth:** Requires authenticated Supabase session; no token passed from browser
- **Endpoints proxied:**
  - `GET /v1/aisp/accounts` → normalized account list
  - `GET /v1/aisp/accounts/:id/transactions?from=&to=&limit=` → normalized transaction list
- **Schema validation:** zod — raw upstream payloads are validated; failing payloads return `502 schema_violation`
- **Error shape:** `{ code, message, upstreamStatus }` — never forwards raw upstream payloads directly
- **No credentials in bundle:** gateway URL and token live only in edge function secrets, not env vars exposed to browser

### Client Service: `src/services/obf/accounts.ts`

- `fetchAccounts(): Promise<OBFAccount[]>` — calls `obf-accounts` edge function
- `fetchTransactions(accountId: string, opts?: { from?: string; to?: string; limit?: number }): Promise<OBFTransaction[]>`
- Both throw on error (caller handles error state)
- Feature flag check lives in the React layer, not here

### Shared Schema (normalized shapes — defined in gateway repo, mirrored here)

```ts
interface OBFAccount {
  id: string;
  displayName: string;
  maskedNumber: string;
  type: "CURRENT" | "SAVINGS" | string;
  currency: string;
  balance: number;
  asOf: string; // ISO 8601
}

interface OBFTransaction {
  id: string;
  accountId: string;
  postedAt: string; // ISO 8601
  amount: number;
  currency: string;
  direction: "credit" | "debit";
  counterparty: string;
  description: string;
  status: "posted" | "pending" | "failed";
}
```

### React Integration

New hook: `useOBFAccounts()` — keyed on `[VITE_OBF_LIVE]`, only calls the edge function when flag is `true`.

Dashboard layout with `VITE_OBF_LIVE=true`:
```
┌─────────────────────────────────────────┐
│  Internal Wallets (existing panel)       │
│  AccountCard | AccountCard | AddAccount  │
├─────────────────────────────────────────┤
│  OBF Accounts (new panel, Providus)       │
│  [name] [maskedNumber] [balance]         │
├─────────────────────────────────────────┤
│  Internal Transactions (existing panel)  │
└─────────────────────────────────────────┘
```

Loading state: OBF panel shows its own skeleton while fetching. Error state: OBF panel shows error inline with retry — does not crash the dashboard.

## Acceptance Criteria

1. With `VITE_OBF_LIVE=true` and gateway reachable, OBF panel shows ≥1 real Providus account
2. Gateway down → OBF panel shows error state (inline, no crash, no stale data)
3. Response payload validated by zod in edge function; failing payloads return `502 schema_violation`
4. No bank credentials, gateway tokens, or upstream URLs in browser bundle
5. Console: zero new errors on `/dashboard` in live mode
6. `VITE_OBF_LIVE=false` (default) → zero network calls to `obf-accounts`
