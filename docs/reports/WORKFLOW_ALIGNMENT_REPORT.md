## What the 3 Sources of Truth Are

1. **Canonical Repository/Branch (`main-protected`)**: The production branch that requires PR approvals and passes all CI checks before merge. This is the authoritative source for deployable code.

2. **CI-Generated Artifacts (GitHub Actions + Build Outputs)**: The build artifacts, test results, and deployment artifacts generated by GitHub Actions workflows. These validate code quality and provide an immutable record of what was built.

3. **External Registry/Documentation (Supabase Edge Functions, Vercel Deployments, Published Docs)**: The deployed Supabase Edge Functions, Vercel deployments, and external documentation that represent the running state of the application. These confirm what is actually running in production.
